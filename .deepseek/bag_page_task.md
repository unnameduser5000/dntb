# 背包页面改造任务

## 目标

将当前战斗界面中的按键编排内容迁移为一个独立的背包页面，并通过 `Tab` 键打开或关闭。

## 需要完成的工作

1. 建立独立背包页面

- 在游戏 HUD 层新增专门的背包页面，不再依赖当前 `BattlePanel` 作为主编辑区域。
- 背包页面打开时覆盖在游戏界面上方。
- `BattlePanel` 从正常游戏界面中移除，不再显示。

2. 使用 `Tab` 控制背包开关

- 按下 `Tab` 打开背包页面。
- 再次按下 `Tab` 关闭背包页面。
- 打开背包时自动暂停游戏内输入和流程。
- 关闭背包时恢复游戏。

3. 区分可编辑状态

- 在营地 / 休息点中，背包页面允许拖拽编辑按键绑定。
- 在战斗场地中，背包页面可以查看，但不可编辑。
- 不可编辑时要明确禁用拖拽与变更操作。

4. 左侧按键槽布局

- 背包页面左侧中上区域显示三行四列按键槽：
  - `Q W E R`
  - `A S D F`
  - `Z X C V`
- 每个槽位代表一个物理按键。
- 每个槽位内可绑定多个行动 token。
- 触发该物理按键时，按槽位内顺序依次执行绑定的 token。

5. 右侧未分配背包区域

- 背包页面右侧显示当前未分配的技能 / token。
- 玩家可以将右侧 token 拖拽到左侧任意按键槽中完成绑定。
- 也需要支持从槽位拖回右侧，表示取消分配。

6. 永久增益展示

- 背包页面左侧下方显示永久增益效果列表。
- 每个永久增益至少显示名称或简短标签。
- 鼠标悬停时显示具体效果说明。

7. 输入与底层数据同步

- 将当前仅支持四个实体键槽的按键编排模型扩展到 `QWER / ASDF / ZXCV` 共 12 个实体槽。
- 保留现有行动 token 解析逻辑，例如 `U / D / L / R / F / TL / TR / J`。
- 物理键槽与行动 token 分层：
  - 物理键槽负责存储顺序；
  - token 负责转换为实际行动。

8. 验证要求

- 确认 `Tab` 开关背包时不会误触发战斗行动。
- 确认休息点可编辑、战斗中不可编辑。
- 确认拖拽分配和取消分配都能正确更新底层数据。
- 确认一个物理键绑定多个 token 时，会按顺序执行。
- 确认永久增益悬停说明可用。

## 相关改动范围

- `scripts/core/PlayerInputService.gd`
- `scripts/core/ActionProgramController.gd`
- `scripts/runtime/KeyProgram.gd`
- `scripts/view/BattleUI.gd`
- `scenes/ui/BattleUI.tscn`
- 可能新增专门的背包页面脚本 / 场景

## 当前约束

- 尽量不要把新的玩法规则继续堆进 `Game.gd`。
- UI 负责展示、拖拽和信号转发。
- 行动执行和规则解析仍保持在现有 core / resolver 边界内。
