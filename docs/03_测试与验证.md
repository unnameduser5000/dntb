# DNTB 测试与验证

## 当前可用烟测

项目已有一个 headless 烟测脚本：

```powershell
godot --headless --path . --script res://scripts/tests/SmokeTest.gd
```

它覆盖的重点包括：

- run 从营地进入第一场战斗。
- 敌人意图和攻击范围预览。
- 玩家行动队列执行和回合推进。
- 世界切片 tavern footprint、酒馆安全区编辑开关，以及出生 tavern NPC 生成/交互/追踪提示。
- 世界切片 NPC 不堵门、交互动作走专门的 `interact` 行动，并且不会在交谈前触发敌方下一轮。
- 统一 actor 交互能力后，验证“可交互怪物”和“可被杀 NPC”都走同一套 `ActorDef` / `ActorState` / `ActionResolver` 规则。
- 世界切片交互框每次只显示一条内容，按任意键关闭；酒馆 actor 贴墙生成，不会再次站到主通路中央。
- 胜利奖励、行动 token 池、按键槽编辑和预览。
- 营地/休息点允许编辑，战斗/Boss 节点锁定编辑。
- 连续移动速度、冲击盾撞击伤害和击退。
- 武器 hook：命中、落空、行动链结束。
- EffectPipeline：伤害复制、移动复制、击退放大、事件反应。
- 遗物奖励跨房间生效并进入存档数据。
- 成就注册、进度、解锁、保存和恢复。

## 本地验证建议

提交前优先跑：

```powershell
godot --headless --path . --script res://scripts/tests/SmokeTest.gd
```

如果本机 Godot 不在 PATH，可以用安装路径直接运行，例如：

```powershell
& "C:\Path\To\Godot.exe" --headless --path . --script res://scripts/tests/SmokeTest.gd
```

手动验证推荐路径：

1. 运行 `res://scenes/game/App.tscn`。
2. 从主菜单开始游戏。
3. 在营地拖动按键槽 token，确认背包页与键位编排刷新正常。
4. 按 `Tab` 或点击右下角 `背包 / Tab` 打开背包，确认游戏暂停，关闭后恢复。
5. 在背包左侧确认 `QWER / ASDF / ZXCV` 三行四列槽位显示正确；当前默认每个键位先显示 `2` 个固定槽位，且左侧整块不会再被挤压成中间窄条。
6. 在右侧确认未分配 token 使用固定 5 列多行布局，token 过多时通过滚动查看。
7. 将右侧未分配 token 拖到左侧槽位，确认只有营地/休息点/酒馆安全屋 footprint 内的可走格可编辑；站到 tavern 互动点上也不应重新锁定。
8. 把鼠标悬停到永久增益和动作 token 上，确认会显示具体 tooltip。
9. 在出生 tavern 安全区内确认只生成一位默认 NPC；当前初始 NPC 是字形 `K` 的 `酒馆掌柜`。走到其相邻格并面向它后，按确认键或物理 `F` 键槽，确认屏幕中下方会出现交互框，显示说话人和世界观文本，并且调试面板里的 tracked actor 相对方向会随玩家位置变化。
10. 在交互框存在时，尝试移动或提交按键槽行动，确认不会推进玩家行动，也不会触发敌人的下一轮；对话框每次只显示一条内容，按任意键会直接关闭，而不是翻到下一句。
11. 进入战斗，确认按键槽锁定。
12. 清空第一房间，选择行动或遗物奖励。
13. 在战后休息点继续调整 token。
14. 进入后续战斗，确认奖励跨房间生效。
15. 打到 Boss 或失败，确认结算和重开流程。

## 新功能测试建议

新增行动：

- 测 token 能否进入槽位。
- 测 `ActionProgramController.build_plan()` 是否生成正确 `ActionInstance`。
- 测 hover 预览是否合理。
- 测实际执行结果和预览一致。

新增武器：

- 测移动碰撞、攻击命中、攻击落空、行动链结束中至少一个 hook。
- 测武器接管默认行为时，默认伤害不会重复结算。
- 测 momentum 或 chain speed 参数是否按预期传入。

新增遗物/修饰器：

- 测 `modify_packets()` 的顺序和 tag 过滤。
- 测 `react_to_event()` 的触发次数和递归深度。
- 测跨房间应用、存档和载入。

新增 UI：

- 测 1280x720 下不遮挡、不溢出。
- 测键鼠操作能触发对应信号。
- 测 UI 只发意图，不直接改核心规则状态。

## 当前验证边界

历史开发日志记录过一次限制：某次环境里 Godot 可执行文件不在 PATH，因此没有运行烟测。后续报告验证结果时要区分：

- 已运行 Godot 烟测。
- 只做了静态阅读或文本搜索。
- 只在编辑器里手动点过流程。

不要把静态检查写成运行验证。
