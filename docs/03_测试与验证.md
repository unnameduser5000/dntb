# DNTB 测试与验证

## 当前可用烟测

项目已有一个 headless 烟测脚本：

```powershell
godot --headless --path . --script res://scripts/tests/SmokeTest.gd
```

它覆盖的重点包括：

- run 从营地进入第一场战斗。
- 敌人意图和攻击范围预览。
- 玩家行动队列执行和回合推进。
- 胜利奖励、行动 token 池、按键槽编辑和预览。
- 营地/休息点允许编辑，战斗/Boss 节点锁定编辑。
- 连续移动速度、冲击盾撞击伤害和击退。
- 武器 hook：命中、落空、行动链结束。
- EffectPipeline：伤害复制、移动复制、击退放大、事件反应。
- 遗物奖励跨房间生效并进入存档数据。
- 成就注册、进度、解锁、保存和恢复。

## 本地验证建议

提交前优先跑：

```powershell
godot --headless --path . --script res://scripts/tests/SmokeTest.gd
```

如果本机 Godot 不在 PATH，可以用安装路径直接运行，例如：

```powershell
& "C:\Path\To\Godot.exe" --headless --path . --script res://scripts/tests/SmokeTest.gd
```

手动验证推荐路径：

1. 运行 `res://scenes/game/App.tscn`。
2. 从主菜单开始游戏。
3. 在营地拖动按键槽 token，并按 WASD 试招。
4. 按 `Tab` 或点击右下角 `背包 / Tab` 打开背包，确认游戏暂停，关闭后恢复。
5. 在背包左侧确认 `QWER / ASDF / ZXCV` 三行四列槽位显示正确，同一槽内 token 为从左到右顺序触发。
6. 将右侧未分配 token 拖到左侧槽位，确认只有营地/休息点/酒馆休息区可编辑。
7. 把鼠标悬停到永久增益条目上，确认会显示具体描述 tooltip。
8. 进入战斗，确认按键槽锁定。
9. 清空第一房间，选择行动或遗物奖励。
10. 在战后休息点继续调整 token。
11. 进入后续战斗，确认奖励跨房间生效。
12. 打到 Boss 或失败，确认结算和重开流程。

## 新功能测试建议

新增行动：

- 测 token 能否进入槽位。
- 测 `ActionProgramController.build_plan()` 是否生成正确 `ActionInstance`。
- 测 hover 预览是否合理。
- 测实际执行结果和预览一致。

新增武器：

- 测移动碰撞、攻击命中、攻击落空、行动链结束中至少一个 hook。
- 测武器接管默认行为时，默认伤害不会重复结算。
- 测 momentum 或 chain speed 参数是否按预期传入。

新增遗物/修饰器：

- 测 `modify_packets()` 的顺序和 tag 过滤。
- 测 `react_to_event()` 的触发次数和递归深度。
- 测跨房间应用、存档和载入。

新增 UI：

- 测 1280x720 下不遮挡、不溢出。
- 测键鼠操作能触发对应信号。
- 测 UI 只发意图，不直接改核心规则状态。

## 当前验证边界

历史开发日志记录过一次限制：某次环境里 Godot 可执行文件不在 PATH，因此没有运行烟测。后续报告验证结果时要区分：

- 已运行 Godot 烟测。
- 只做了静态阅读或文本搜索。
- 只在编辑器里手动点过流程。

不要把静态检查写成运行验证。
